# 初回実装 要求内容

## 概要

プロダクト要求定義書・機能設計書・技術仕様書に基づき、タスク管理CLIツール `task` の
コアとなる最小動作可能なバージョン（v1）を実装する。

---

## 実装スコープ

### 対象機能

v1 として以下をすべて実装する。

| 機能 | コマンド | 優先度 |
|------|---------|--------|
| タスクCRUD | `task add` `task list` `task show` `task edit` `task done` | 必須 |
| 自動スコアリング | `task next` `task list`（スコア順） | 必須 |
| Gitコンテキスト連携 | `task add`（ブランチ自動紐付け）`task next`（コンテキスト加点） | 必須 |
| 依存関係管理 | `task block` `task unblock` | 必須 |
| スヌーズ・ピン固定 | `task snooze` `task pin` `task unpin` | 必須 |
| プリセット設定 | `.taskrc` の `preset` 読み込みとスコア重み適用 | 必須 |
| TODO同期 | `task sync` `task sync --dry-run` | 必須 |
| データエクスポート | `task export --format json\|md` | 必須 |

### 対象外（v2以降）

- チームでのタスク共有・アサイン機能
- インストールスクリプト（`install.sh`）
- GUI・Web UI

## 受け入れ条件

### 動作要件

- `make build` でバイナリが生成されること
- `make test` が全件グリーンで通ること（カバレッジ: scoring 100%、その他 80%以上）
- `make lint` が通ること
- 全コマンドの応答時間が 200ms 以内であること

### 機能要件

- すべての必須コマンドが動作すること
- `task next` がスコア計算結果に基づいて正しいタスクを返すこと
- `.taskrc` のプリセット設定が反映されること
- `~/.task/tasks.db` が自動生成され、パーミッションが `600` であること
- gitリポジトリ外で実行しても `task add` などの基本操作が動作すること

### 非機能要件

- `NO_COLOR=1` でカラーなし出力に切り替わること
- `--json` フラグで JSON 出力に切り替わること
- `--verbose` フラグでスコア計算のデバッグ情報が出力されること

---

## 制約事項

- 言語：Go 1.22以上
- データ保存：SQLite（`modernc.org/sqlite`）のみ。外部サービス不使用
- CGO不使用（クロスコンパイル対応のため）
- macOS / Linux で動作すること
